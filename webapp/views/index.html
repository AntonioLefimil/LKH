LKH Version 2.0.7 (November 2012)

Abstracto


LKH is an effective implementation of the Lin-Kernighan heuristic for solving the traveling salesman problem.Computational experiments have shown that LKH is highly effective. Even though the algorithm is approximate, optimal solutions are produced with an impressively high frequency. LKH has produced optimal solutions for all solved problems we have been able to obtain; including a 85,900-city instance (at the time of writing, the largest nontrivial instance solved to optimality). Furthermore, the algorithm has improved the best known solutions for a series of large-scale instances with unknown optima, among these a 1,904,711-city instance (World TSP). The DIMACS TSP Challenge (2000) provided many benchmark instances. Their sizes range from 1,000 to 10,000,000 cities. LKH currently holds the record for all instances with unknown optima. The lengths of the current best tours are tabulated here. David Soler has provided 126 asymmetric instances with known optima. LKH's performance on these instances is reported here.

Parámetros

PASOS
Nombre Descripcíon Valor
CANDIDATE_SET_TYPE Specifies the candidate set type. ALPHA is LKH's default type. It is applicable in general. The other three types can only be used for instances given by coordinates. The optional suffix PURE for the DELAUNAY type specifies that only edges of the Delaunay graph are used as candidates. Default: ALPHA.
INITIAL_TOUR_ALGORITHM Especifica el algoritmo para la obtención de un recorrido inicial. Por defecto: WALK
EXTRA_CANDIDATES Número de candidato adicional bordes para ser añadido al conjunto de candidatos de cada nodo. Su tipo de conjunto de candidatos puede especificarse utilizando la palabra clave TYPE EXTRA_CANDIDATE_SET_. El número entero puede ser seguido por la palabra clave SIMÉTRICA, lo que significa que estos bordes candidatos adicional es que complementarse de tal manera que cada uno de ellos se asocia tanto con sus dos nodos finales. Por defecto: 0.
KICKS (Entero) Especifica el número de veces para "desconectar" un recorrido encontrado por Lin y Kernighan. Cada patada es una Tiro al azar K-swap. Sin embargo, si TIROS es cero, entonces estrategia especial de patadas LKH, WALK, se utiliza en lugar de él. Por defecto: 1.
KICK_TYPE (Entero) Especifica el valor de K para un saque de K-swap aleatoria (una extensión de la doble puente movimiento). Si KICK_TYPE es cero, entonces estrategia especial patadas del LKH, paseo, se utiliza. Por defecto:
SUBPROBLEM_SIZE El número de nodos en una división del problema original en subproblemas. La división se realiza de acuerdo a el tour dada por SUBPROBLEM_TOUR_FILE. El valor 0 significa que no hay división se hace. Por defecto los subproblemas se determinan mediante la subdivisión de la gira en segmentos de igual tamaño
SUBPROBLEM_TOUR_FILE (String) Especifica el nombre de un archivo que contiene un tour que debe utilizarse para dividir el problema original en subproblemas. El número aproximado de nodos en cada uno viene dado por SUBPROBLEM_SIZE. El recorrido está dada por una lista de enteros que proporcionan la secuencia en la que los nodos se visitan en el tour. El recorrido termina con un -1
STOP_AT_OPTIMUM Especifica si se interrumpe un desplazamiento, si la longitud de recorrido se hace igual al óptimo.Por defecto: SÍ.
SUBGRADIENT Especifica si los valores de pi deben ser determinados por la optimización subgradiente. Por defecto: SÍ.
SUBSEQUENT_PATCHING Especifica si los parches se utiliza para los movimientos siguientes el primer paso en una secuencia de se mueve. Por defecto: SÍ.
RESTRICTED_SEARCH Especifica si se usa la siguiente técnica de búsqueda poda: El primer borde para ser rota en un movimiento no debe pertenecer a la mejor gira actualmente solución. Cuando no hay solución recorrido es conocido, no debe pertenecer a la expansión mínimo 1-árbol. Por defecto: SÍ.
RUNS (Entero) El número total de carreras.
BACKBONE_TRIALS (Entero) El número de ensayos columna vertebral en cada ejecución. Por defecto: 0
MAX_CANDIDATES (Entero) El número máximo de candidato bordes para ser asociada a cada nodo. El número entero puede ser seguido por la SYMMETRIC palabra clave, lo que significa que el conjunto de candidatos es ser complementado tal que cada borde candidato se asocia con ambos de sus dos nodos finales. Si MAX_CANDIDATES es cero los conjuntos de candidatos se componen de los bordes representados en el CANDIDATE_FILEs, la INITIAL_TOUR_FILE, la INPUT_TOUR_FILE, la SUBPROBLEM_TOUR_FILE, y los MERGE_TOUR_FILEs. Por defecto: 5.
MAX_SWAPS (Entero) Especifica el número máximo de permutas (voltea) permitidos en cualquier búsqueda de una mejora de gira.Por defecto: valor de dimensión.
NONSEQUENTIAL_MOVE_TYPE (Entero) Especifica el tipo de movimiento no secuencial para ser utilizado. Un valor K> = 4 significa que los intentos se ha intentado mejorar un recorrido por no secuenciales k-opt mueve en 4 k K. Nótese, sin embargo, que el efecto depende de las especificaciones de PATCHING_A y PATCHING_B. Por defecto: valor de (+ MOVE_TYPE PATCHING_A + PATCHING_B - 1).
POPULATION_SIZE (Entero) Especifica el tamaño máximo de la población en el algoritmo genético de LKH. Por defecto: 0.
PRECISION (Entero) La precisión interna en la representación de distancias transformadas: d [i] [j] = PRECISION * c [i] [j] + pi [i] + pi [j], donde d [i] [j], c [i] [j], pi [i] y pi [j] son ​​todos integral. Por defecto: 100 (que corresponde a 2 cifras decimales).
SEED (Entero) Especifica la semilla inicial para la generación de números aleatorios. Por defecto: 1.
TIME_LIMIT Entero) Especifica un límite de tiempo en segundos para cada ejecución. Por defecto: valor de DBL_MAX.
TRACE_LEVEL (Entero) Especifica el nivel de detalle de la salida dada durante el proceso de solución. El valor 0 significa una cantidad mínima de salida. Cuanto mayor sea el valor es el más información se dado. Por defecto: 1.
GAIN23 Especifica si se utiliza la función Gain23, Por defecto: SÍ.
GAIN_CRITERION Especifica si se utiliza Lin y Kernighan del criterio de la ganancia. Por defecto: SÍ.
BACKTRACKING Especifica si un retroceso K-opt movimiento es para ser utilizado como el primer paso en una secuencia de movimientos (cuando = TIPO DE MOVIMIENTO). Por defecto NO.
MAX_TRIALS (Entero) El número máximo de los ensayos en cada ejecución. Por defecto: número de nodos (dimensión, que figuran en el archivo del problema).
MOVE_TYPE (Entero) Especifica el tipo movimiento que se utilizará para todos los movimientos siguientes el primer movimiento en una secuencia de movimientos. El valor K> = 2 significa que un movimiento K-opt se va a utilizar. El valor 0 significa que todos los movimientos son del mismo tipo (K = MOVE_TYPE). Por defecto: 0.
